# v0 Provenance Checklist

H0 is a Vercel v0 + AWS Databases challenge. The app already has Vercel + AWS
database proof. The remaining submission risk is v0 provenance.

This local Codex environment does not currently expose `V0_API_KEY`, so the repo
cannot honestly claim a new automated v0 generation run from this machine. Use
one of the verified evidence paths below before final submission.

## Acceptable Evidence To Attach

- v0 chat URL showing the dashboard or component generation history.
- Screenshot of the v0 prompt history and preview.
- GitHub branch or PR created by v0.
- Terminal output from a real v0 "Add to Codebase" or `npx v0@latest add`
  command.
- A short demo-video segment showing v0 prompt, generated UI, and final merged
  app.

## Prompt Pack For a Real v0 Pass

Use this prompt in v0 if a fresh artifact is needed:

```text
Improve a Next.js finance dashboard for an H0 Vercel + AWS Databases hackathon.
The app is Archon H0: SMB finance intelligence with document intake, seven-agent
run ledger, P&L, cash, sales performance, purchase concentration, payroll
controls, source citations, Ask Archon Q&A, and AWS DynamoDB evidence.

Keep the interface dense and work-focused, not a marketing landing page.
Use a restrained CFO dashboard style, visible sponsor-stack proof, and an
activity history section showing persisted REPORT and ACTIVITY records.
Do not mention Gemini, Google, Azure, Teams, or Foundry in the visible app.
```

Recommended follow-up prompt:

```text
Polish the judge path for a 2-minute demo. Make the first viewport show:
Vercel + AWS, Run Finance Close, db_mode=aws-dynamodb, P&L revenue,
EBITDA, sales goal attainment, and links to /api/report, /api/evidence,
and /api/history. Preserve existing API contracts and TypeScript types.
```

## How To Record It In Devpost

Use wording only if there is real evidence:

```text
We used Vercel v0 to iterate on the Archon H0 dashboard and judge path, then
merged the generated UI improvements into a Next.js App Router app deployed on
Vercel. The backend persists finance reports and activity records to AWS
DynamoDB.
```

If no real v0 artifact exists, use a safer statement:

```text
The project is a Vercel-hosted Next.js app backed by AWS DynamoDB. The repository
includes a v0 prompt/provenance checklist, but the final submission should not
claim v0 generation unless a real v0 artifact is attached.
```

## Internal Rule

Do not fabricate a v0 chat URL, screenshot, prompt history, generated branch, or
CLI output. A weaker honest claim is better than a disqualifying false one.
